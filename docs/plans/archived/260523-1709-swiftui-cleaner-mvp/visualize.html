SwiftUI Mac Cleaner MVP

A native macOS SwiftUI MVP for safe disk analysis, cleanup review, project artifact purging, installer cleanup, and Trash-first removal. The plan deliberately avoids privileged optimization, realtime system monitoring, Touch ID/PAM changes, background auto-cleaning, and permanent deletion.

Phase 2 [L]: Implement scan core and Analyze

Add models, rules, scanner, cancellation, summaries, tests, and an Analyze view for user-owned roots.

Phase 3 [L]: Implement Clean, Purge, Installers, and Trash flow

Add cleanup rules, selection de-duplication, review UI, Finder reveal, and explicit move-to-Trash behavior.

Phase 4 [M]: Safety polish, tests, and release readiness

Harden permission diagnostics, settings, UI states, tests, logging, documentation, and final QA evidence.

Risks And Questions

Primary risks

  • macOS permissions may block useful scan roots.
  • Large directory scans can freeze the UI if work reaches the main actor.
  • Cleanup mistakes can destroy user trust even when files are recoverable.
  • Parent and child selections can double-count or double-act on paths.
  • Symlinks can escape the intended scan root.

Open questions

  • Minimum supported macOS version is not yet fixed.
  • Distribution model is not yet decided: local, unsigned, Developer ID, or sandboxed.
